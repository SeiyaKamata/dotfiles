# バグ修正起点スキル

## 役割
ユーザーからのバグ報告（症状）を起点に、**再現確認 → 原因の一次特定 → `bug-report.md` への記録**までを行う。
`/fix`（ループ内のテスト失敗の修復。起点は `test-report.md`）や `/hotfix`（本番 tag 起点の緊急対応）とは起点が異なる。実際のコード修正・回帰テスト・レビュー・コミットは行わず、`/fix`・`/test`・`/review`・`/commit` に委ねる。

## 設計方針：コンテキストフリー
このスキルは会話履歴に依存しない。入力はすべて `.specs/<feature>/` と実際のコード調査から得る。コールドで `/bugfix <feature>` を叩いても動く。

## 入力（すべて .specs / 調査から。会話に依存しない）
- `.specs/<feature>/bug-report.md`（存在すれば。前回までの調査結果。コールド再開時のベースドラフト）
- ユーザーから伝えられるバグの症状・再現手順・期待/実際の挙動
- 実際のコード（Grep/Read で疑わしい箇所を調査）

## 進め方

### Step 1: 引数チェック
- `$ARGUMENTS[0]`（feature）が未指定なら「使い方: /bugfix <feature>」を表示して終了する。

### Step 2: コンテキスト収集とモード分岐
- `.specs/<feature>/bug-report.md` があれば読み込み、その内容をベースドラフトとしてコールド再開する（既に埋まっている項目は聞き直さない）。
- 無ければ新規受領モードとして Step 3 へ進む。
- 会話履歴には依存しない。

### Step 3: バグ報告の受領
症状・再現手順・期待する挙動・実際の挙動をユーザーから聞き取る。
症状または再現手順が不足している場合は、不足している項目を質問して補完する。

### Step 4: 再現の確認
再現手順が揃ったら、実際に再現を試みる。
- 再現できた場合 → Step 5 へ進む
- 再現できない場合 → `bug-report.md` に「再現できず」と記録し、ユーザーに追加情報を求めて**ここで停止する**（修正工程へ勝手に進まない）

### Step 5: 原因の一次特定
再現が確認できている間、Grep/Read でコードを調査し、原因の疑わしい箇所（ファイル・関数）を絞り込む。
**根本原因の確定診断と実際のコード修正は行わない**（それは `/fix` に委ねる）。

### Step 6: bug-report.md 保存（レビューゲート）
以下の必須項目が埋まっているかを自己チェックしてから `.specs/<feature>/bug-report.md` に保存する：
- [ ] 症状
- [ ] 再現可否
- [ ] 疑わしい箇所（再現できた場合）
- [ ] 想定する修正範囲（再現できた場合）
- [ ] 回帰テストの観点（再現できた場合）

再現できなかった場合は「再現できず」を記録した時点で Step 6 を満たしたものとして扱い、Step 7 には進まず停止する。

### Step 7: 次ステップ提示
`bug-report.md` を保存したら、末尾の「次ステップ提示」の定型ブロックを提示する。次スキルは自動起動しない。

## 出力フォーマット

### bug-report.md（`.specs/<feature>/bug-report.md`）
`/fix` が読むバグ調査記録。`/bugfix` が毎回作成／更新する（コールド再開時はベースドラフトとして読み直す）。

```markdown
# バグ報告: [機能名]

## サマリ
- 症状: [一文]
- 再現可否: 再現OK / 再現できず
- 期待する挙動: [...]
- 実際の挙動: [...]

## 再現手順
1. ...
2. ...

## 疑わしい箇所（一次特定・確定診断は /fix に委ねる）
- <path>:<line> — [なぜ疑わしいか]

## 想定する修正範囲
- [触るファイル・関数の見当]

## 回帰テストの観点（実行は /test に委ねる）
- [この症状が再発しないことを確認する観点]
```

## 完了条件
`.specs/<feature>/bug-report.md` の保存と次ステップ提示をもって完了とし、次スキルを自動起動しない。
再現できなかった場合は、「再現できず」の記録とユーザーへの追加情報要求をもって完了とする（次ステップ提示は行わない）。

## 次ステップ提示
単体起動で完了したら、次の定型ブロックを**コードフェンスで囲まず**プレーンテキストで出力し、次スキルは自動起動せずユーザーの実行を待って終了する。

```
✅ バグ調査完了 — .specs/<feature>/bug-report.md
OK：/fix <feature>
NG：/bugfix <feature>（再調査）
```
