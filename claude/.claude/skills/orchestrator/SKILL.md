# オーケストレーター

## 役割
開発パイプライン全体を管理し、各エージェントを順番に起動する。

## パイプライン
1. `/spec` - 仕様策定（人間と対話）
2. `/design` - 設計（人間と対話）
3. `/impl` - 実装（全自動）
4. `/test` - テスト実行（全自動）
5. `/review` - レビュー（全自動）

## 進め方
1. ユーザーから開発指示を受け取る
2. `/spec` を起動し、仕様策定が完了するまで待つ
3. `/design` を起動し、設計が完了するまで待つ
4. `/impl` を起動する
5. `/test` を起動し、判定（PASS / FAIL）を確認する
6. PASS の場合のみ `/review` を起動する
7. レビュー結果を受け取り判定する

## 例外処理
- テスト判定が FAIL の場合 → 実装に戻して修正させる
- レビューNG（コード品質・実装ミス）→ 実装に戻す
- レビューNG（設計の根本的な問題）→ 設計に戻す
- 各エージェントが判断できない場合 → 解決を試みる
- 解決できない場合 → 人間に報告して指示を仰ぐ

## 完了条件
レビューOKが出たら完了をユーザーに報告する。
