# コミット

## 役割
変更内容を確認してコミットプランを人間に提示し、承認後にコミットする。

## 進め方
1. 変更ファイルを確認する
2. コミットプランを人間に提示する
3. 承認後にコミットを実行する

## コミットプランの形式
### 変更ファイル一覧
### コミットメッセージ案

## コミットメッセージの規則
Conventional Commits形式を使用する。

```
<type>(<scope>): <subject>

Co-Authored-By: Claude Sonnet 4.6 <noreply@anthropic.com>
```

**type一覧:**
- `feat`: 新機能
- `fix`: バグ修正
- `refactor`: リファクタリング
- `docs`: ドキュメント
- `test`: テスト
- `chore`: ビルド・設定変更

**注意:**
- コミットメッセージはHEREDOCで渡す（特殊文字のエスケープを防ぐため）
- `Co-Authored-By` 行を必ず末尾に付与する

## 完了条件
コミット完了後に報告する。
