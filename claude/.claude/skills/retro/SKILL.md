# /retro - 介入分析レトロスペクティブ

セッションログから「人間が介入した箇所」を抽出し、次回は介入なしで進められるようにCLAUDE.md / memory / skill を更新する。

## 実行手順

### Step 1: ログを収集する

引数があればその日付、なければ今日を対象にする。

Bashツールで以下を実行する（`CLAUDE_SKILL_DIR` と `ARGUMENTS` は実際の値に置き換える）：

```
chmod +x "<CLAUDE_SKILL_DIR>/scripts/collect_logs.sh"
bash "<CLAUDE_SKILL_DIR>/scripts/collect_logs.sh" --skill-dir "<CLAUDE_SKILL_DIR>" <ARGUMENTS>
```

出力が `NO_LOGS_FOUND` の場合は「本日のセッションが見つかりませんでした」と伝えて終了する。

### Step 2: 介入パターンを検出する

ログを読み、以下の **4つのシグナル** に該当する箇所を抽出する。

1. **否定・修正** — ユーザーが「いや」「違う」「そうじゃなくて」「ダメ」等でClaudeの出力や方向性を否定した
2. **具体的指示の上書き** — Claudeが何かを提案/実行した後、ユーザーが「〇〇して」「〇〇にして」と別の方法を指示した（Claudeが自力で正しい判断をすべきだったケース）
3. **繰り返し説明** — ユーザーが同じ意味の説明を2回以上している
4. **手戻り** — Claudeが作業した後にやり直し・取り消しが発生した

**重要:**
- 「良かったこと」は分析しない。介入がなかった箇所はスキップする
- 介入が0件なら「介入なし。改善不要です」と伝えて終了する
- 各介入について「ユーザーの発言」と「その直前のClaudeの発言・行動」をセットで記録する

### Step 3: 介入を分類しアクションを生成する

検出した各介入を以下の4カテゴリに分類し、**実行可能なドラフト** を生成する。

| 分類 | 原因 | アクション |
|---|---|---|
| **A. プロジェクト知識不足** | プロジェクト固有のルール・構造・慣例を知らなかった | CLAUDE.mdに追記するテキストのドラフト |
| **B. ユーザーの好み** | ユーザー固有の作業スタイル・好みを知らなかった | memoryファイルに保存する内容のドラフト |
| **C. 繰り返しパターン** | 毎回同じ手順を手動で指示している | skill化の概要（何をするskillか） |
| **D. 判断ミス** | 知識はあったが判断を誤った | feedbackメモリに保存する内容のドラフト |

**ドラフト生成のルール:**
- CLAUDE.md追記: プロジェクトの既存CLAUDE.mdのスタイルに合わせた1〜3行のテキスト
- memory保存: frontmatter付きのmemoryファイルの全文（type: feedback or user or project）
- skill化: 「/spec で詳細化しましょう」の提案のみ（retro内では実行しない）
- 既にCLAUDE.mdやmemoryに同等の内容がある場合はスキップする

### Step 4: アクション案を提示する

以下のフォーマットで出力する。

```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
  介入分析レトロ - YYYY-MM-DD
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

検出した介入: N件

---
### 介入 1: [簡潔な説明]
プロジェクト: xxx
何が起きたか: [ユーザーがこう言った → Claudeはこうしていた]
なぜ介入が必要だったか: [分類A/B/C/D + 理由]
アクション:
  → [CLAUDE.mdに追記 / memoryに保存 / skill化候補]: 「具体的なテキスト」
---

### 介入 2: ...
...

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
実行しますか？ (全部 / 番号指定 / スキップ)
```

### Step 5: 承認後に即実行する

ユーザーの応答に応じて実行する:
- **「全部」** → 全アクションを順に実行
- **番号指定**（例:「1, 3」）→ 指定のみ実行
- **「スキップ」** → 何もしない

実行方法:
- **CLAUDE.md追記** → 該当プロジェクトのCLAUDE.mdファイルを直接編集する
- **memory保存** → memoryファイルを作成し、MEMORY.mdにインデックスを追加する
- **skill化** → 「/spec で詳細化しましょう」と伝えて終了する
